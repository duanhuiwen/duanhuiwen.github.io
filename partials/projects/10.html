Rovio Data analysis and visualization System

2013.10 - 2014.5 | Espoo, Finland

Key words: Java, AngularJS, Highcharts, RIAK, Linux, Cloud Computing, Data Visualization.

Rovio has 250 Million monthly users and therefore Cloud Services must undergo extensive performance testing. For Rovio, such services include gamer logins, saved sessions, video feeds, advertisements, web visitors, online shopping and many more.

There is a dedicated performance testing system and a test cluster to simulate heavy user loads against Cloud Services. Performance testing data is collected from testing Clusters and moved to central place for analysis. This data is real performance data of Rovio Services. The analysis of the results could be much streamlined and improved to help us to identify the performance bottlenecks of Rovio Services. Visualization of the data is also part of this project.


Project goals:

The project goal is an independent data analysis system and visualization service. The service enables software developers, test engineers and other users to:

  • Visualise the essential metrics of performance tests.
  • Compare the results to previous results, e.g. in order to determine whether the performance of an application degraded when a new feature was added.
  • Identify bottleneck components in Rovio Services
  • Helps Rovio to improve the performance of the Production Environment

The system is extendable and Rovio could continue developing it further.

Tools and technology:

A cloud server environment, a distributed database, a set of APIs and the input data format will be provided by Rovio. In this project, I was mainly responsible for front-end development. The service used:

  • Java SE 7 in backend.
  • Accept JSON input from cluster test nodes.
  • Save the analysed data in a permanent storage (RIAK).
  • AngularJS and highcharts for front end real time monitoring.
  • Run on a default server installation of Ubuntu Linux 12.04.
